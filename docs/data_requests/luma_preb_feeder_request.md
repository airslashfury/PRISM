## 1. What PRISM is and why we're asking

PRISM models Puerto Rico's physical systems — power, water, roads, telecom, emergency response —
as one interconnected system, to make the *consequences* of infrastructure decisions visible
(which communities lose power, water, and hospital access when a given substation fails, and which
investments most reduce that risk).

PRISM is built on authoritative public data wherever it exists (HIFLD substations and transmission,
Census, USGS, FEMA flood zones, and PREPA's own published generation feed). **One relationship,
however, is not published anywhere: which distribution feeder serves which customer/load.** Today
PRISM approximates it with a spatial heuristic (a Voronoi/voltage-hierarchy proxy, confidence
0.4–0.7 per link). Every downstream resilience and investment figure is honestly labeled "Proxy"
because of it.

A single dataset from LUMA would replace that approximation with measured assignments and raise
PRISM's resilience rankings, consequence estimates, and equity-weighted investment portfolios from
*Proxy* to *Authoritative/Modeled*. This is the highest-leverage data gap in the entire model.

## 2. Primary request — distribution feeder topology (LUMA)

The connectivity model linking substations → feeders/circuits → served loads. Either as a GIS export
or a connectivity table. The minimum useful fields:

| Field | Purpose in PRISM |
|---|---|
| Feeder / circuit ID | Primary key for the feeder |
| Source substation (name or ID) | The `FEEDS` edge: substation → feeder |
| Feeder geometry (centerline) **or** served service-area polygon | Spatial extent of the circuit |
| Nominal voltage (kV) | Voltage-hierarchy validation |
| List of served delivery points, or polygon of served parcels/meters | The `POWERS` edge: feeder → load |
| Critical-load flags (hospital / water / lifeline feeders), if maintained | Prioritization of consequence scoring |

Even a **coarse** version — feeder service-area polygons without per-meter detail — would be a major
upgrade over the current proxy. Aggregated/anonymized is fine; PRISM needs *topology*, not individual
customer records.

## 3. Secondary request — reliability statistics (LUMA, filed with PREB)

These replace national-default constants currently used as placeholders:

| Field | Replaces (PRISM Estimated constant) |
|---|---|
| **SAIDI / SAIFI per feeder** (or per region) | Flat 33.6 outage-hours/yr national default |
| Historical outage / restoration records by feeder (post-Maria, Fiona, Apr-2024 blackout) | Hand-curated municipio-level backtest ground truth |
| Any **value-of-lost-load (VOLL)** or outage-cost study for PR | $5/kWh national default ($2,389/person 30-yr) |

The outage/restoration history is also what would let us validate PRISM's hazard model against real
events at the feeder level instead of the coarse municipio level we use now.

## 4. What this unlocks (so the value is concrete)

With items in §2, these PRISM outputs move from **Proxy → Authoritative/Modeled**:
`graph.relationships` (FEEDS/POWERS) · per-substation downstream population & facility counts ·
cascade and scenario resilience scores · community-resilience equity index · VOLL exposure ·
the ILP investment portfolio · scenario comparisons. That is nine model outputs from one dataset.

## 5. Format, delivery, and data handling

- **Format:** GeoPackage / Shapefile / File-GDB / PostGIS dump for geometry; CSV/XLSX for tables.
  Any CRS is fine (PRISM reprojects to EPSG:32161).
- **Delivery:** secure transfer of your choice (SFTP, cloud bucket, physical media).
- **Handling:** PRISM mirrors data locally, versioned and checksummed. We will honor any
  aggregation, anonymization, redaction, or non-redistribution terms LUMA/PREB specify, and will
  cite the source and vintage on every figure derived from it (PRISM already displays per-table
  provenance and confidence tiers publicly).
- **Scope:** this is a public-interest resilience-planning request; no individual customer
  identification is needed or wanted.
